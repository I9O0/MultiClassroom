# 高校智慧教室管理系统

## 1、课设项目介绍
该系统拥有四种角色，实现高校教室资源的数字化管理，涵盖教室信息维护、预约、设备报修等核心功能：

👴系统管理员：
- 查看全校区教室使用统计（可用教室数、预约量、报修量等）
- 管理所有教学楼/教室信息（新增、编辑、禁用）
- 管理用户信息（学生、教师、教室管理员）
- 处理所有教室的预约审核与设备报修

👩教室管理员：
- 查看所有教学楼的教室状态（使用中、空闲、维护中）
- 维护所有教室的设备信息（投影仪、音响等配置）
- 处理所有教学楼的设备报修申请
- 审核所有学生/教师的教室预约请求
- 管理所有教室的新增、编辑、状态更新（启用/禁用）

👨‍🏫教师：
- 查询可用教室（支持按教学楼、日期、设备配置筛选，仅可查看J3、J4教学楼）
- 查看教室多媒体设备详情
- 预约符合教学需求的教室
- 管理个人预约记录（查看、取消、签到/签退）

👨‍🎓学生：
- 查询开放的学生专用教室及可用时间段（仅可查看J1、J2教学楼）
- 预约自习或小组讨论用教室
- 提交教室设备故障报修申请
- 查看个人预约及报修记录


## 2、开发工具
- 后端：IntelliJ IDEA
- 前端：VS Code 或 WebStorm


## 3、项目技术
- 后端框架：Spring Boot、MyBatis、MyBatis-Plus
- 前端技术：Vue、Element Plus、CSS、JavaScript、Axios
- 数据库：MySQL


## 4、环境：
JDK≥1.8、Maven、Node.js ≥16、MySQL ≥8.0


## 5、硬件环境：
Windows 或者 Mac OS(不挑剔)


## 配置
1. 在项目根目录或 `doc` 中找到 `smart_classroom_sys.sql`，执行脚本初始化数据库（含表结构及测试数据）。

2. 后端文件夹 `Smart_Classroom_System_backend`：
   - 修改 `src/main/resources/application.properties` 文件，配置本地端口号、数据库地址、账号及密码。
   - 启动 `SpringbootApplication` 主类运行后端服务。

3. 前端文件夹 `Smart_Classroom_System_frontend`：
   - 进入文件夹目录，打开控制台执行 `npm install` 安装依赖。
   - 执行 `npm run serve` 启动前端服务，通过终端提示的地址访问系统。

4. 初始登录信息（数据库对应表中可查看更多）：
   - 系统管理员：用户名 `admin`，密码 `123456`
   - 教室管理员：用户名 `manager1` 等，密码 `123456`
   - 教师：用户名 `teacher1` 等，密码 `123456`
   - 学生：用户名 `stu1` 等，密码 `123456`
